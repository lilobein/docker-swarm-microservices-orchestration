# Docker Swarm Microservices Orchestration
##### все кроме сервисов джавы написано руками  ᕦ(ò_óˇ)ᕤ
Данный проект представляет собой практическую реализацию оркестрации микросервисной архитектуры с использованием Docker Swarm. Цель проекта — демонстрация полного цикла развертывания распределенного приложения: от контейнеризации отдельных сервисов до управления кластером и мониторинга. ლ(｀ー´ლ)

## Архитектура проекта
Проект включает в себя многоконтейнерное приложение, состоящее из нескольких микросервисов:

- Gateway Service — фасад (API Gateway), единая точка входа для всех клиентских запросов, маршрутизирующий их к соответствующим внутренним сервисам
- Session Service — сервис аутентификации и управления пользовательскими сессиями
- Hotel Service — сервис для работы с данными об отелях (каталог, наличие номеров, информация)
- Booking Service — сервис управления бронированиями (создание, изменение, отмена)
- Payment Service — сервис обработки платежей и финансовых транзакций
- Loyalty Service — сервис программы лояльности (скидки, бонусы, статусы пользователей)
- Report Service — сервис сбора статистики и формирования отчетов о работе системы
- RabbitMQ — брокер сообщений для асинхронного взаимодействия между сервисами
- Nginx — проксирование запросов
- Portainer — просто для меня （ ^_^）o自自o（^_^ ）

## Технологический стек
- Docker — контейнеризация микросервисов
- Docker Compose — локальная оркестрация контейнеров
- Docker Swarm — кластеризация и распределенное развертывание
- Vagrant — автоматизация создания виртуальных машин
- Nginx — обратный прокси для маршрутизации трафика
- Portainer — визуальный мониторинг и управление кластером

## Ключевые особенности
- Контейнеризация микросервисов — созданы оптимизированные Dockerfile для каждого компонента системы
- Локальное развертывание — с помощью Docker Compose обеспечена корректная работа всех сервисов с пробросом необходимых портов
- Ифраструктура как код — Vagrantfile автоматизирует создание трех виртуальных машин (менеджер и два воркера)
- Кластеризация — развернут Docker Swarm кластер с распределением сервисов по узлам
- Безопасность — настроен nginx как обратный прокси, скрывающий внутренние сервисы от прямого доступа
- Мониторинг — интегрирован Portainer для визуализации состояния кластера

## Процесс развертывания
- Локальная сборка и тестирование микросервисов через Docker Compose
- Публикация образов на Docker Hub
- Создание виртуальных машин через Vagrant
- Инициализация Docker Swarm кластера
- Развертывание стека сервисов на кластере
- Настройка проксирования и проверка работоспособности через Postman тесты

## Тестирование
Корректность работы системы подтверждается прохождением всех Postman тестов, проверяющих:
- Доступность сервисов через nginx прокси
- Корректное взаимодействие между микросервисами
- Сохранение состояния сессий
- Отсутствие прямого доступа к внутренним сервисам

## Результаты
В ходе выполнения проекта:
- Созданы оптимизированные Docker образы микросервисов
- Развернут отказоустойчивый кластер из трех узлов
- Реализована изоляция внутренних сервисов через прокси
- Настроен визуальный мониторинг через Portainer
- Подтверждена работоспособность системы автоматическими тестами
